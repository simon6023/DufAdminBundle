Configure Index lists

The Indexable annotation allows you to configure the list view of your CRUD.

To use this feature, import IndexableAnnotation in your entity : 

	
use Duf\AdminBundle\Annotations\IndexableAnnotation as Indexable;

Several property types are supported :

  • Default
  • Boolean
  • ManyToOne

Global parameters :

  • index_column : boolean | required
    Set to true, the list table will have this column
    Set to false, the column will not be displayed
  • index_column_name : string | required
    The header label of the column
  • prefix : string | optional
    Text to display before the value of the field, in the list table
  • suffix : string | optional
    Text to display after the value of the field, in the list table

Default

Valid for following property types : string, text, integer, float, datetime.

Definition :

		
* @Indexable(index_column=true, index_column_name="Title")
Boolean

Definition :

		
* @Indexable(index_column=true, index_column_name="Enabled", boolean_column=true)

Parameters :

  • boolean_column : boolean | required
    Set true to use a boolean column (will display 'Yes' or 'No')

ManyToOne

Definition :

		
* @IndexableAnnotation(index_column=true, index_column_name="Category", relation_index="title")

Parameters :

  • relation_index : string | required
    the property of the inverse side of the relation. Must be a string property, as it will be displayed in the list table.
